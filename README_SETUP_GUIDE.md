# دليل إعداد وتشغيل مشروع File Reader على VS Code

## 1. نظرة عامة على المشروع

المشروع المرفق هو تطبيق ويب كامل (Full-Stack) مصمم ليكون واجهة مطعم (Damas Food Restaurant) يعرض قائمة طعام تفاعلية. يستخدم المشروع التقنيات التالية:

*   **الواجهة الأمامية (Client-side):** React.js مع Vite كأداة بناء، TypeScript، و Tailwind CSS للتصميم.
*   **الواجهة الخلفية (Server-side):** Express.js مع TypeScript.
*   **إدارة الحالة:** Zustand (مُشار إليه من خلال `useCart` في `client/src/store/cart.tsx`).
*   **قاعدة البيانات (محتمل):** Drizzle ORM و PostgreSQL (مُشار إليهما في `package.json`، ولكن التخزين الافتراضي حاليًا هو في الذاكرة).

يهدف المشروع إلى توفير تجربة مستخدم سلسة لعرض المنتجات وإدارة سلة التسوق، مع دعم للغات متعددة (العربية، الإنجليزية، الفرنسية).

## 2. المتطلبات الأساسية

قبل البدء، تأكد من تثبيت الأدوات التالية على جهازك:

*   **Node.js:** يفضل الإصدار 18 أو أحدث. يمكنك تنزيله من [nodejs.org](https://nodejs.org/).
*   **npm أو pnpm:** مدير حزم Node.js. يأتي npm مع Node.js، ويمكنك تثبيت pnpm باستخدام `npm install -g pnpm`.
*   **Visual Studio Code (VS Code):** محرر الأكواد البرمجية. يمكنك تنزيله من [code.visualstudio.com](https://code.visualstudio.com/).

## 3. إعداد المشروع في VS Code

اتبع الخطوات التالية لإعداد وتشغيل المشروع:

### أ. فتح المشروع

1.  افتح VS Code.
2.  اذهب إلى `File > Open Folder...` (أو `ملف > فتح مجلد...`).
3.  استعرض المجلد الذي قمت باستخراج `File-Reader-1.zip` إليه، ثم حدد المجلد `File-Reader-1` واضغط `Open`.

### ب. تثبيت التبعيات

المشروع يستخدم `pnpm` كمدير للحزم. افتح الطرفية المدمجة في VS Code (يمكنك ذلك من `Terminal > New Terminal` أو `الطرفية > طرفية جديدة`).

1.  انتقل إلى مجلد المشروع الرئيسي `File-Reader-1`:
    ```bash
    cd /home/ubuntu/file_reader_project/File-Reader-1
    ```
2.  قم بتثبيت جميع التبعيات (Dependencies) باستخدام `pnpm`:
    ```bash
    pnpm install
    ```
    إذا لم يكن `pnpm` مثبتًا، يمكنك تثبيته أولاً باستخدام `npm install -g pnpm` ثم أعد تشغيل الأمر `pnpm install`.

### ج. تشغيل التطبيق

المشروع يتكون من جزأين: الواجهة الأمامية (Client) والواجهة الخلفية (Server). ستحتاج إلى تشغيل كليهما بشكل منفصل في وضع التطوير.

1.  **تشغيل الواجهة الخلفية (Server):**
    في طرفية VS Code جديدة (يمكنك فتح طرفية جديدة من `Terminal > New Terminal`):
    ```bash
    pnpm run dev
    ```
    سيقوم هذا الأمر بتشغيل خادم Express.js باستخدام `tsx`.

2.  **تشغيل الواجهة الأمامية (Client):**
    في طرفية VS Code أخرى جديدة:
    ```bash
    pnpm run dev:client
    ```
    سيقوم هذا الأمر بتشغيل خادم تطوير Vite للواجهة الأمامية. بمجرد التشغيل بنجاح، ستظهر لك رسالة في الطرفية تحتوي على رابط (عادةً `http://localhost:5000` أو ما شابه) يمكنك فتحه في المتصفح لرؤية التطبيق.

    **ملاحظة:** إذا واجهت مشكلة في تشغيل `dev:client`، قد تحتاج إلى التأكد من أن `vite` يعمل بشكل صحيح. قد يكون هناك تعارض في المنافذ إذا كان المنفذ 5000 مستخدمًا بالفعل.

## 4. ما ينقص المشروع والنواقص المحتملة

بناءً على فحص الكود، إليك بعض الملاحظات حول ما ينقص المشروع أو يمكن تحسينه:

*   **قاعدة البيانات:** حاليًا، يستخدم المشروع `MemStorage` (في `server/storage.ts`)، وهو تخزين مؤقت في الذاكرة. هذا يعني أن أي بيانات تُضاف أو تُعدّل ستُفقد عند إعادة تشغيل الخادم. على الرغم من وجود `drizzle-orm` و `pg` (PostgreSQL) في التبعيات، إلا أن التكامل الفعلي مع قاعدة بيانات PostgreSQL غير موجود في `storage.ts` الافتراضي. لتفعيل قاعدة بيانات دائمة، ستحتاج إلى:
    *   إعداد قاعدة بيانات PostgreSQL (محليًا أو على خادم).
    *   تعديل `server/storage.ts` لاستخدام `drizzle-orm` للاتصال بقاعدة البيانات بدلاً من `MemStorage`.
    *   تشغيل `pnpm run db:push` لإنشاء الجداول في قاعدة البيانات بناءً على `drizzle.config.ts` و `shared/schema.ts`.

*   **مسارات الـ API (API Routes):** ملف `server/routes.ts` فارغ حاليًا. هذا يعني أنه لا توجد مسارات API معرفة في الواجهة الخلفية للتعامل مع عمليات مثل إضافة طلبات، إدارة المستخدمين، أو أي تفاعلات أخرى تتطلب حفظ البيانات. ستحتاج إلى إضافة هذه المسارات في `server/routes.ts` لتمكين الواجهة الأمامية من التفاعل مع الواجهة الخلفية بشكل كامل.

*   **المصادقة والتفويض (Authentication/Authorization):** على الرغم من وجود `passport` و `express-session` في التبعيات، إلا أن منطق المصادقة والتفويض الفعلي غير مطبق في الكود الحالي. صفحة `Admin.tsx` موجودة، ولكن لا يوجد نظام تسجيل دخول أو حماية للمسارات. ستحتاج إلى تطبيق هذا المنطق لتأمين الأجزاء الإدارية من التطبيق.

*   **إدارة الصور:** يتم استيراد الصور مباشرة في `client/src/data/menu.ts` من مسار `@assets`. تأكد من أن هذه المسارات صحيحة وأن الصور موجودة في المجلد `attached_assets`.

## 5. الخلاصة

المشروع هو نقطة بداية جيدة لتطبيق مطعم، ولكنه يتطلب بعض العمل لإضافة وظائف الواجهة الخلفية الكاملة وتكامل قاعدة البيانات. باتباع الخطوات المذكورة أعلاه، يمكنك تشغيل الواجهة الأمامية والخلفية في وضع التطوير والبدء في استكشاف الكود وإضافة الميزات المطلوبة.
